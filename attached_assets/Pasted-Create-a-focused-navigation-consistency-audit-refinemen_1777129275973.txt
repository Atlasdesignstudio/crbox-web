Create a focused navigation consistency audit + refinement pass for the authenticated client portal, covering both top navigation layers across all portal pages and both breakpoints.

This is not a redesign.
This is a consistency, modularity, and polish audit so the two nav bars feel identical in quality, spacing, sizing, behavior, and implementation across the full client portal.

## Context
The client portal has two navigation layers:

1. the main top navbar/header  
   - logo
   - account/user area
   - CTA
   - hamburger/menu behavior on mobile

2. the portal tab bar / secondary nav
   - Dashboard
   - Mis Paquetes
   - Mis Facturas
   - Mi Cuenta

These generally behave well, but they need a full audit to ensure they are truly consistent across every portal page and not drifting slightly in:
- height
- spacing
- paddings
- icon sizing
- text sizing
- active-state thickness/position
- distances between elements
- mobile vs desktop behavior
- implementation structure

## Objective
Audit and refine the two portal nav layers so that across:
- Dashboard
- Mis Paquetes
- Mis Facturas
- Mi Cuenta

they:
- look visually identical where they should
- use the same spacing system
- use the same sizing system
- behave consistently
- feel polished and deliberate
- are implemented in the most maintainable/modular way possible within the current project structure

## What to check
### 1. Cross-page consistency audit
Compare the two nav layers across all authenticated portal screens and verify whether they are truly identical in:
- total height
- top/bottom padding
- left/right padding
- gap between icon and label
- text size/weight
- icon size
- active-state underline/border treatment
- vertical alignment
- spacing between items
- account area spacing
- logo/header positioning
- CTA placement
- responsive behavior

If there are even small pixel-level drifts between pages, normalize them.

### 2. Main navbar consistency
Audit the main top navbar/header across all portal pages, on both desktop and mobile.

Check:
- logo placement
- overall header height
- account/user block alignment
- CTA size and spacing
- dropdown trigger spacing
- hamburger placement
- internal padding
- visual rhythm
- whether one page has slightly different spacing/sizing than another

Refine so the header is identical and stable everywhere.

### 3. Portal tab bar consistency
Audit the secondary tab bar across all portal pages, on both desktop and mobile.

Check:
- tab height
- item spacing
- icon/label alignment
- active-state line/border
- overflow behavior on mobile
- horizontal scroll behavior if present
- side padding
- visual centering
- tap target size
- consistency of selected-state treatment

Make sure the tab bar looks and behaves like the same component everywhere.

### 4. Desktop and mobile best practices
Apply best practices for both breakpoints.

For desktop:
- spacing should feel balanced and premium
- nav layers should feel stable and aligned
- active states should be crisp and consistent

For mobile:
- header and tab bar should still feel intentional, not squeezed
- tap targets should be comfortable
- scroll/overflow behavior should be stable
- the portal tabs should remain easy to read and interact with

### 5. Modularity / implementation audit
Check whether these nav bars are currently duplicated or slightly diverging across files.

If possible within the current structure, improve maintainability so both nav layers are driven by a more consistent shared structure/pattern across all authenticated pages.

This does not mean introducing a new framework or rebuilding the whole app.
It means:
- reducing drift
- making repeated structures more consistent
- avoiding page-by-page variation
- keeping the implementation easier to maintain

If full modularization is not appropriate in the current architecture, at minimum ensure the markup/classes/styles are normalized across all portal pages.

### 6. No regression
Do not break:
- current page navigation
- active-tab behavior
- user dropdown behavior
- hamburger behavior
- desktop layout
- mobile layout

This is a refinement and consistency pass, not a rebuild.

## Deliverable
Audit and refine the two top navigation layers across the full authenticated client portal so they are visually and behaviorally consistent across all screens and breakpoints.

In the final report, briefly state:
1. whether any inconsistencies were found between portal pages
2. what was normalized in the main navbar
3. what was normalized in the portal tab bar
4. whether any modular/shared-structure improvements were applied
5. confirmation that desktop and mobile both remain stable and consistent
