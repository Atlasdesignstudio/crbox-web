# CRBOX Product Brain Master Database Builder
# Costa Rica Courier / Product Category / Estimated Tax / Risk Rules

Build the CRBOX Product Brain database.

This is a source-of-truth data task only. Do not implement UI yet. Do not redesign the quote flow yet. Do not change calculator math yet.

The goal is to create a complete, structured, AI-friendly local database that CRBOX can later use across:
- Public Cotizar
- Public Calculadora
- Client portal Cotizaciones/Solicitudes
- Internal sales/admin quote calculator
- Product URL “Analizar”
- Quote response email
- Portal quote response
- Reminder email
- Quote-related AI/chat flows

This Product Brain must help classify products, estimate tax category, detect risk, detect manual review needs, detect restricted/forbidden products, and generate customer-friendly explanations.

IMPORTANT:
This is not an official customs liquidation system.
All rates are local estimated references for quote UX.
Every record must use source: "local_estimated".
Final taxes may vary by CIF, official classification, exchange rate, Aduanas review, permits, notes, and product specifics.

Do not connect to external APIs.
Do not use live TICA, Hacienda, SAC, Ar@ncelNet, ATENA, SIAA, RIMM, or VUCE integrations.
Design the data so it can later migrate to SQLite/Postgres or official API validation.

==================================================
RATE FORMAT
==================================================

Use decimal format for totalEstimatedRate:

0.14 = 14%
0.15 = 15%
0.20 = 20%
0.24 = 24%
0.29 = 29%

Do NOT store 29 if the calculator expects 0.29.

UX simplified estimate:
- DAI 0% + IVA 13% + Ley 6946 1% = ~14%
- DAI 1% + IVA 13% + Ley 6946 1% = ~15%
- DAI 6% + IVA 13% + Ley 6946 1% = ~20%
- DAI 10% + IVA 13% + Ley 6946 1% = ~24%
- DAI 15% + IVA 13% + Ley 6946 1% = ~29%

If uncertain:
- totalEstimatedRate: null
- estimatedRange: "Variable"
- automaticEstimateAllowed: false or true with range, depending on category
- manualReviewRequired: true if risk/regulation/ambiguity is high

==================================================
CUSTOMER-FACING LANGUAGE RULES
==================================================

Customer messages must be friendly Spanish prose.

Do NOT show customers:
- confidence: high
- DAI: 15
- riskFlags
- local_estimated
- HS code
- manual_review_required
- technical metadata

Use messages like:
“Detectamos que este producto pertenece a Calzado. Esta categoría suele pagar aproximadamente 29% en impuestos. El monto final puede variar según material, tipo de suela y revisión aduanera.”

For manual review:
“Este producto requiere revisión antes de cotizarlo automáticamente. Puede necesitar permisos, validación especial o manejo de transporte.”

For restricted:
“Este producto puede pertenecer a una categoría restringida. CRBOX debe revisarlo antes de confirmar si puede transportarse o importarse por courier.”
