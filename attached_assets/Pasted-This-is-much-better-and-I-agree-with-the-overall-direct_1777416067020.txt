### 1. Clarify scope around package linking
The task currently describes strong auto-match / assisted match / manual fallback, but also says admin tool changes are out of scope.

Please make the scope explicit:

- In scope now
  - storing expected_tracking_number
  - showing the expected-package block in the portal

- Out of scope for this task
  - actual admin-side matching workflows

### 2. Strengthen the pendiente_compra_crbox customer copy
This state is correct, but the customer should understand clearly what it means.

Please use clearer, friendly copy such as:

> “CRBOX comprará este producto en tu nombre y te contactará con los siguientes pasos cuando corresponda.”

This should feel reassuring and actionable, not empty.

### 3. Keep the tracking-number field explicitly optional
For the pendiente_compra_cliente state, the tracking number input should feel helpful but optional.

Recommended helper copy:

> “Si ya tienes el número de seguimiento, agrégalo aquí para ayudarnos a relacionar el paquete más fácilmente cuando llegue.”

Do not make this feel like a blocked step or a required action.

### 4. Make linked_package_id clickable if feasible
If a package is already linked, show it as a read-only relationship, but make it navigable if possible rather than plain text only.

Example:
- Paquete relacionado: #12345 → clickable link to the package view or packages page

### 5. Improve the CTA explanations for the two customer intents
This is very important.

The two options should not appear as generic actions only. Each one needs a short, user-friendly explanation directly under the CTA title so the customer understands the operational difference immediately.

Please integrate short explanatory copy like this into the portal intent-selection UI:

#### Comprar con CRBOX
CRBOX puede encargarse de todo el proceso de compra por ti. Solo necesitarás realizar por adelantado el pago correspondiente al producto o productos cotizados, para que podamos gestionarlo en tu nombre.

#### Lo compraré por mi cuenta
Tú realizas la compra directamente y luego subes al portal la factura y la información necesaria del pedido, para que CRBOX pueda procesarlo y darle seguimiento cuando llegue a Miami.

Important:
- keep the copy short, clear, and user-friendly
- avoid legalistic wording
- make the operational difference immediately obvious
- the copy should feel intuitive and reassuring, not bureaucratic

These explanations should live inside the CTA card/module design itself, so the choice is self-explanatory before the customer clicks.

### 6. Keep the “No continuar” action visually secondary
This should remain available, but low emphasis. It should not visually compete with the two main intent paths.
