Create a focused mobile navigation refinement pass for the authenticated hamburger menu in the CRBOX client portal.

This is a new standalone prompt.
It is not a redesign of the portal.
It is a mobile-only usability and hierarchy refinement for the menu/drawer.

## Problem observed
On mobile, the hamburger menu currently mixes:
- generic site navigation items
- authenticated client-portal items

but the portal-specific options do not feel visually distinct enough from the general website tabs.

Also, the drawer/menu does not scroll properly, so lower actions like Cerrar Sesión can be cut off and become difficult or impossible to access on smaller screens.

The menu should feel more intentional, more readable, and more clearly structured for a logged-in portal user.

## Objective
Refine the mobile hamburger menu so that:

- authenticated client portal options are more clearly identifiable than generic website navigation
- the drawer has a better information hierarchy
- the logged-in user block feels intentional and better separated
- lower menu actions are always reachable
- the whole menu feels cleaner, more premium, and more usable on mobile

## What to improve
### 1. Clearer hierarchy between generic site links and portal links
The mobile drawer currently lists both:
- generic public-site navigation
- authenticated portal navigation

These should not feel visually equivalent.

Without redesigning the overall system, make the portal-specific section feel more clearly identifiable through:
- grouping
- spacing
- separators
- hierarchy
- section labeling if appropriate
- stronger containment of the logged-in/account-related area

The user should immediately understand:
- which links belong to the website in general
- which links belong to their personal portal/account area

### 2. Improve the logged-in identity block
The block with:
- user name
- casillero number

should feel more intentional and better integrated into the drawer hierarchy.

It should help anchor the authenticated area and make the portal navigation feel more contextualized.

### 3. Fix scrolling behavior
This is critical.

The mobile drawer must become properly scrollable so that:
- all items are accessible
- lower actions like Cerrar Sesión never get cut off
- the menu remains usable on smaller heights and narrower mobile devices

The drawer should support full vertical scrolling when content exceeds viewport height.

### 4. Improve spacing and readability
Refine:
- vertical spacing between sections
- spacing between rows
- divider rhythm
- icon/text alignment
- readability of grouped navigation

The result should feel less like a long flat list and more like a clearly structured mobile navigation surface.

### 5. Preserve the current design system
Important constraints:
- do not redesign the entire menu
- do not introduce a new visual language
- do not change brand colors, typography system, or icon system
- do not turn this into a different component pattern

Use the current CRBOX system, but make the menu feel:
- clearer
- more polished
- more mobile-native
- more premium
- more usable

## Functional requirements
- Menu/drawer must scroll vertically when content exceeds the viewport
- Cerrar Sesión must always be reachable
- Portal links must feel more identifiable than generic website links
- Generic site links and authenticated portal links must be visually separated more clearly
- Desktop behavior must not be regressed

## Deliverable
Refine the mobile hamburger menu/drawer so the authenticated portal area is more clearly distinguished and the full menu is scrollable and usable on mobile.

In the final note/report, briefly state:
1. how hierarchy between generic links and portal links was improved
2. how the logged-in account block was refined
3. how scroll/accessibility of the drawer was fixed
4. any mobile viewport edge cases that were specifically handled
