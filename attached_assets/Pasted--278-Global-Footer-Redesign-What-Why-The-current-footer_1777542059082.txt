#278 - Global Footer Redesign

## What & Why
The current footer is repeated across many HTML files, feels visually under-designed, contains outdated or incomplete contact information, uses broken legal links, and does not reflect the same level of polish as the rest of the CRBOX experience.

This task is a footer-only redesign and systemization pass across:
- all public pages
- all client portal pages

The goal is to:
- redesign the footer so it feels more premium, intentional, and modern
- correct contact/hours/location information using the Contact page as source of truth
- make legal links stop behaving incorrectly
- centralize the footer into one maintainable implementation
- ensure the footer is truly designed for mobile, not just technically responsive

This is not a broader redesign of the site or portal.

## Technical approach — centralize the footer
Create a single reusable footer implementation.

### Preferred approach
Create js/footer.js that injects the full footer into a placeholder:
html <div id="site-footer"></div> 

Then replace all repeated static <footer>...</footer> blocks with that placeholder and include:
html <script src="js/footer.js?v=1"></script> 
near the end of each page.

### Requirements
- One source of footer markup
- One source of footer behavior
- Easy to update later without editing many files
- Must work across both public pages and portal pages
- Do not break existing page spacing; adjust wrapper spacing per page only if needed

### Important
Do not create two divergent footer systems unless there is a real structural reason.  
Default to one shared implementation.

## Mobile is a first-class requirement
This footer must not simply collapse. It must feel intentionally designed for mobile.

### Mobile requirements
- stack sections in a clear and deliberate order
- recommended order:
  1. brand
  2. quick links
  3. contact
  4. locations/hours
  5. legal bottom area
- generous vertical spacing between groups
- comfortable touch targets
- readable font sizes
- no cramped lists
- no horizontal overflow
- legal links should not feel dropped awkwardly at the end
- contact rows should be easy to tap and scan
- location information should remain digestible on small screens

### Success criteria on mobile
It should feel like a footer designed for mobile, not a desktop footer that barely survived responsiveness.
