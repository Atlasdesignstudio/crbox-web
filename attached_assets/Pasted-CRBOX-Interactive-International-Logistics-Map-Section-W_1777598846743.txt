CRBOX — Interactive International Logistics Map Section

What & Why

Add a new premium, interactive visual section to the CRBOX Servicios page, placed directly below the “Nuestros servicios / Soluciones a tu medida” section.

The purpose is to visually communicate that CRBOX does more than casillero/courier: it also offers international logistics services for air, maritime, land, LCL, FCL, consolidated cargo, full containers, export courier, and regional transport.

The section should make users immediately understand that CRBOX connects major international origins with Costa Rica through different logistics routes.

This should feel modern, trustworthy, and visually impressive — not like a generic static paragraph.

⸻

Section Concept

Create a section with an animated world map or abstract global route visualization showing international logistics routes converging into Costa Rica.

It does not need to use Google Maps. In fact, avoid embedding Google Maps unless necessary. The ideal solution is a custom, lightweight, animated visual map using HTML/CSS/SVG/JS.

The visual should show routes from the regions CRBOX serves:

* Estados Unidos / Miami
* México
* Centroamérica
* Panamá
* Sur América
* Europa
* Medio Oriente
* Lejano Oriente / Asia
    China, Indonesia, Taiwán, Japón, Korea, Vietnam, entre otros

All routes should visually connect toward Costa Rica.

⸻

Placement

Place this new section inside servicios.html, directly after the existing section titled:

“Nuestros Servicios” / “Soluciones a tu medida”

Do not redesign the whole page. Preserve the current CRBOX design system, spacing, typography, colors, header, footer, and existing content structure.

⸻

* Premium

The result should look like a polished logistics dashboard visual, not like a basic infographic.

Interaction Requirements

The section should include lightweight interactivity:

* Default state shows all routes subtly animated toward Costa Rica.
* Hovering a route chip highlights only that route.
* Clicking a route chip keeps that route active.
* The selected route updates a description panel.
* There should be a “Todos” option to show all routes again.
* On mobile, the map should remain readable and the chips should stack or scroll horizontally.
* Avoid interactions that break accessibility or depend only on hover.
* Keyboard focus states should be visible.

⸻

Animation Requirements

Add subtle, polished motion:

* Route lines should softly animate as if cargo is moving toward Costa Rica.
* Origin points should pulse subtly.
* Costa Rica destination point should be visually emphasized.
* Hovering a chip should create a smooth route highlight.
* The card should feel alive but not distracting.
* Use CSS animations or lightweight vanilla JavaScript.
* Avoid heavy libraries unless already included.

⸻

Suggested UI Layout

Desktop:

* Two-column layout.
* Left side: label, headline, paragraph, service highlights.
* Right side: large animated map card.
* Below the map or inside the card: route chips and selected route details.

Mobile:

* Stack copy above map.
* Map card full width.
* Chips in horizontal scroll or stacked grid.
* Description panel below the chips.

Technical Requirements

* Implement in servicios.html.
* Add CSS either in the existing page <style> block or in css/styles.css if reusable.
* Add JavaScript in main.js or inline only if minimal and page-specific.
* Do not remove existing services.
* Do not change current header/footer behavior.
* Keep Tailwind 2.2.19 and Font Awesome usage consistent with the current site.
* Do not introduce fake prices, fake delivery promises, or unsupported claims.
* The section should be content-realistic and based only on the logistics services listed above.
* Make sure the layout is responsive.
* Make sure the map still looks good if JavaScript fails; default should show all routes and static copy.
