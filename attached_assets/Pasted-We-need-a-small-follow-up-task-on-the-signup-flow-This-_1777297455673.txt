## Issue 1 — Business cédula jurídica must be enforced in frontend
We already confirmed the backend is permissive and currently accepts many formats, including lengths and even alphanumeric values.

That is not the desired business rule.

For the business account flow, we now want the frontend to enforce the expected current business input standard:

- cédula jurídica only
- 10 numeric digits only
- no letters
- no shorter or longer values

### What to implement
In the business registration form only:

- validate the cédula jurídica field before submission
- accept only numeric input
- require exactly 10 digits
- show a clear inline error if the value is invalid
- the error should explain the expected format in plain Spanish

### UX requirement
Do not make this feel harsh or technical.
It should feel clear and helpful.

Use something like:
- “Ingresa una cédula jurídica válida de 10 dígitos.”

If helpful, allow the user to type hyphens visually, but normalize to digits before validating and before sending.  
If that adds unnecessary risk, keep the input numeric-only and validate the final value as exactly 10 digits.

### Important
- Do not change the backend contract
- Do not add future alphanumeric support
- Do not change personal account validation
- This is a frontend business-rule constraint only
