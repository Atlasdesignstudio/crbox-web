Context:
CRBOX currently has a basic mobile dropdown menu across public pages and client portal pages. I want this upgraded into a polished mobile side navigation drawer that feels modern, app-like, and premium, using CRBOX branding.

Important:
This is an aesthetic and UX revamp only. Do not remove, rename, or break any existing navigation tabs, links, login states, dashboard tabs, portal actions, logout behavior, or backend-related logic. Every tab/link currently visible in each page’s mobile menu must still exist after the redesign.

Desired behavior:
- On mobile, tapping the hamburger button opens a side drawer.
- The drawer should slide in from the left.
- The drawer should cover around 80% of the screen width, similar to the reference.
- The remaining 20% of the screen should show the current page behind a dark blurred overlay.
- The background/page behind the drawer should be dimmed and blurred.
- A large “X” close button should appear at the top right area of the overlay/drawer experience, similar to the reference.
- Tapping the X closes the drawer.
- Tapping outside the drawer closes the drawer.
- Pressing Escape closes the drawer.
- When the drawer is open, body scrolling should be locked.
- When the drawer closes, scroll should be restored.
- The hamburger icon should not simply expand a block below the header anymore.

Visual direction:
- Use CRBOX orange as the main brand accent.
- The drawer should feel clean, modern, and app-like.
- Use a strong top profile/header area inspired by the reference, but adapted to CRBOX:
  - Public/logged-out state: CRBOX logo, short line such as “Compra en USA y recibe en Costa Rica”, and account actions.
  - Logged-in portal state: user name if available, casillero number if available, and a small avatar/icon treatment.
- Drawer background can be white or very subtle warm off-white.
- Add CRBOX orange highlights for active states, icons, CTA pills, and hover/focus states.
- Menu rows should be tall, touch-friendly, and visually separated with subtle dividers.
- Icons should remain consistent with the current Font Awesome setup.
- Active/current page should be clearly highlighted.
- Keep the UI professional and trustworthy, not childish.
