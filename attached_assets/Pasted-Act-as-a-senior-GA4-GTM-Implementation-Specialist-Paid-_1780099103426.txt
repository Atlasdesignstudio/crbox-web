Implementation requirements:

A. UTM Capture + Persistence

Implement a safe UTM attribution layer in js/analytics.js or the existing centralized analytics utility.

Capture these parameters from the URL:
- utm_source
- utm_medium
- utm_campaign
- utm_content
- utm_term
- gclid
- fbclid

Store safely in sessionStorage.

Recommended structure:
- crbox_utm_last_touch
- crbox_utm_first_touch

Rules:
- First-touch should only be set if empty.
- Last-touch should update when new attribution params are present.
- Do not store PII.
- Do not store full URLs.
- Do not store arbitrary query params.
- Do not capture tokens, emails, phone numbers, IDs, tracking numbers, invoice numbers, package IDs, addresses, or invoice data.
- Do not capture raw form values.
- Attribution persistence must never break the website if sessionStorage is unavailable.
- If sessionStorage is blocked or unavailable, tracking should gracefully continue without attribution data.
- Store only approved attribution keys.
- Sanitize and truncate attribution values where appropriate to avoid unexpected payload size or unsafe values.

B. Attach Attribution to Events

Add safe attribution parameters to CRBOX.track event payloads if attribution data is available.

Recommended safe attribution parameters:
- utm_source
- utm_medium
- utm_campaign
- utm_content
- utm_term
- gclid_present: true/false
- fbclid_present: true/false
- attribution_touch: first_touch / last_touch / both_available / none

Important:
Before adding new attribution parameters to event payloads, check the existing registered parameter policy in docs/tracking-plan.md and docs/analytics-taxonomy.md.

If adding utm_source, utm_medium, utm_campaign, utm_content, utm_term, gclid_present, fbclid_present, or attribution_touch requires updating the documented GA4 custom dimension registry, document the required GA4/GTM updates clearly.

Do not silently add parameters that GTM/GA4 will not read. If they are added to dataLayer but not yet configured in GTM/GA4, mark them clearly as:

“dataLayer-ready, GTM/GA4 configuration pending”

Do not send raw gclid or raw fbclid to GA4 unless the existing measurement plan explicitly approves it. Prefer boolean presence flags for analytics payloads.

Do not send full landing page URLs or referrer URLs as event parameters.

Apply attribution consistently through the centralized analytics utility so existing and future CRBOX.track events inherit the same attribution behavior.
