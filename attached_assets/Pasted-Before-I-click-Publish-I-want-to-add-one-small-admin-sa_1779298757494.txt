Task:
Fix the quote request handling in the sales/admin panel when a quoted item cannot be processed, cannot be shipped, or needs extra review.

Current issue:
When a quote item is marked as “not available” or equivalent, the UI hides fields such as final price or quote details. That part makes sense because those fields should not be required if the item cannot be processed.

However, when trying to submit/send the response, the system still validates those hidden fields as required. This blocks submission even though the selected status means those fields should not be needed.

Expected behavior:
- If the item is approved / can be processed, price and required quote fields should remain required.
- If the item needs review / confirmation, the fields needed to explain or complete the quote should remain available and validation should match that state.
- If the item cannot be processed, hidden pricing/detail fields should NOT be required.
- The admin should be able to send the response without filling final price fields for an item that cannot be processed.
- The customer-facing response should clearly explain the situation instead of producing a missing-form error.

Also review the wording:
The current wording sounds too much like CRBOX is selling a product with “available / not available” inventory. CRBOX is providing a shipping, courier, and logistics service. The language should feel more service-oriented, clear, and helpful.

Please improve the status labels and supporting copy.

Use short internal labels for the card/status UI, but keep the meaning clear for the sales/admin team.

Suggested status language:

1. Positive status — the item can move forward

Internal short label options:
- “Se puede enviar”
- “Podemos gestionarlo”
- “Se puede cotizar”

Meaning:
The item can be handled by CRBOX and the quote can move forward. Pricing/details should be required.

Customer-facing meaning:
“Podemos gestionar el envío de este producto.”

2. Middle status — the item may be possible, but needs review

Internal short label options:
- “Requiere revisión”
- “Por confirmar”
- “Necesita validación”

Meaning:
The item is not rejected, but CRBOX needs to verify something before confirming the final quote. This could be because of product type, dimensions, weight, category, restrictions, taxes, documentation, or missing information.

Customer-facing meaning:
“Necesitamos revisar algunos detalles antes de confirmar si podemos gestionarlo.”

3. Negative status — the item cannot be processed

Internal short label options:
- “No se puede enviar”
- “No podemos gestionarlo”
- “No aplica para envío”

Meaning:
CRBOX cannot process or ship this item under the current conditions. Pricing fields should not be required because there is no final quote to calculate.

Customer-facing meaning:
“No podemos gestionar el envío de este producto bajo las condiciones actuales.”
