#309 - Replace Old Chat Bubble With Modern Floating AI Prompt Bar

## What & Why

Replace the current floating orange circular chat bubble on the public CRBOX website with a modern horizontal floating AI prompt bar.

The current bubble pattern feels outdated, generic, and associated with low-quality support bots. It looks like a traditional customer-service chat widget rather than an intelligent navigation and assistance layer.

The new experience should feel closer to a modern AI assistant entry point: subtle, premium, useful, always available, and clearly inviting the user to ask anything about CRBOX.

The goal is to make the assistant feel like part of the site experience, not a third-party chat bubble.

This should be implemented across all public pages, not inside the authenticated dashboard unless already present and explicitly needed.

Public pages include:
- index.html
- servicios.html
- como-funciona.html
- tarifas.html
- calculadora.html
- contacto.html
- afiliate.html
- login.html, only if it does not interfere with the login form

Do not modify private/dashboard pages:
- dashboard.html
- mis-paquetes.html
- mis-facturas.html
- mi-cuenta.html

## Current State

There is currently a floating round orange chat button near the bottom-right of the page, next to the back-to-top button.

This should be removed or replaced visually.

The back-to-top button can remain, but its layout must be adjusted so it does not collide with the new AI prompt bar.

## Desired Experience

Instead of a circular chat bubble, add a floating horizontal assistant bar near the bottom center of the viewport.

The bar should look like a polished AI search/chat input, similar to a modern assistant command bar.

Visual reference:
- Rounded pill-shaped horizontal input
- Soft shadow
- White or very light background
- Subtle border
- Small attachment/chat icon on the left
- Placeholder text:
  “Pregúntanos lo que quieras”
  or
  “Pregunta sobre envíos, tarifas o cómo comprar”
- A circular send/arrow button on the right
- Premium micro-interactions on focus, hover, and open state

It should feel modern, intelligent, and trustworthy — not like a legacy chatbot.

## Core Behavior

Default collapsed state:
- Show a floating horizontal prompt bar at the bottom center.
- The bar should be visible enough to invite interaction, but not block core content.
- On mobile, it should span most of the viewport width with safe margins.
- On desktop, it should have a comfortable max-width.

When the user clicks/focuses the bar:
- Expand into a larger assistant panel above the bar or transform the bar into an expanded composer.
- Show quick prompt suggestions.
- Keep the interaction lightweight and elegant.
- Do not navigate away.
- Do not open a new tab.

Quick prompt examples:
- ¿Cómo funciona CRBOX?
- ¿Cuánto cuesta traer un paquete?
- ¿Qué puedo comprar en USA?
- ¿Cómo uso mi casillero?
- ¿Dónde está mi paquete?
- ¿Qué productos requieren permisos?
- Quiero cotizar un envío internacional

The prompt suggestions should appear as clickable chips/cards.

When clicking a suggestion:
- Fill the input with that suggestion.
- Optionally show a lightweight response placeholder or route to the relevant page/section if no real AI backend exists.
- Do not fake an AI answer if there is no real AI/chat backend.
- If no backend is connected, show a truthful helper state:
  “Estamos preparando esta asistencia inteligente. Mientras tanto, te llevamos a la información más útil.”
- Then provide relevant shortcut links:
  - Cómo funciona → como-funciona.html
  - Calculadora → calculadora.html
  - Tarifas → tarifas.html
  - Contacto / WhatsApp → contacto.html or existing WhatsApp link
  - Afíliate gratis → afiliate.html

Important:
Do not create fake intelligence.
Do not invent real-time tracking, pricing, customs validation, or chatbot responses unless the backend exists.
The assistant can be positioned as a smart navigation and guidance layer for now.

## Design Direction

The component should feel:
- Modern
- Intelligent
- Minimal
- Premium
- Friendly
- Fast
- Trustworthy
- Integrated into CRBOX

Avoid:
- Generic chatbot bubble look
- Large intrusive popups
- Fake typing indicators
- Mascot-style bot design
- Overly playful UI
- Floating orange circle as the main interaction
- Dark opaque chat windows that feel third-party

Use CRBOX design language:
- Orange #FF6B00 as accent
- White / light gray surface
- Soft border
- Soft shadow
- Rounded pill shape
- Clean typography
- Subtle gradients only if tasteful
- Font Awesome icons if already used
- Smooth transitions

## Suggested UI Structure

Create a reusable HTML component near the bottom of public pages, before the closing body tag:

Wrapper:
- .ai-prompt-shell

Collapsed bar:
- .ai-prompt-bar
- Left icon: paperclip, sparkles, message, or search-style icon
- Placeholder/input:
  “Pregúntanos lo que quieras”
- Right action button:
  Arrow icon or paper-plane icon

Expanded panel:
- .ai-prompt-panel
- Small title:
  “Asistente CRBOX”
- Supporting text:
  “Haz una pregunta o elige una opción para encontrar rápidamente lo que necesitas.”
- Quick prompt chips/cards
- Optional shortcut area:
  “Accesos rápidos”
- Close/minimize control

Recommended visual behavior:
- The panel expands upward from the bar.
- The bar remains anchored at the bottom.
- Use transform, opacity, and max-height transitions.
- On mobile, panel should be full-width with safe margins and not exceed viewport height.
- The body should not lock scroll unless absolutely necessary.

## Functional Requirements

- Remove or hide the existing round orange chat bubble from public pages.
- Keep or reposition the back-to-top button so it does not overlap with the new prompt bar.
- Add the new assistant prompt bar to all public pages consistently.
- The bar should open/close smoothly.
- The input should be focusable.
- Suggestion chips should be clickable and keyboard accessible.
- Escape key should close the expanded panel.
- Clicking outside should close the expanded panel only if the input is empty.
- Focus states must be visible.
- Use aria-expanded, aria-controls, and appropriate labels.
- Respect prefers-reduced-motion.
- Do not interfere with existing mobile nav, forms, calculators, or page scripts.

## No-JS Fallback

If JavaScript fails:
- The collapsed bar can remain visible as a link-like helper.
- Quick links should still be accessible if possible.
- Nothing should block page content.

## Technical Requirements

- Use vanilla JavaScript only.
- No external chat widget libraries.
- No external AI APIs unless already implemented in the project.
- No backend changes.
- Add reusable CSS to css/styles.css.
- Add reusable JS to js/main.js or the existing main JS file used by public pages.
- If adding repeated markup manually to each public page, keep it identical.
- Prefer a small reusable HTML snippet pattern if the project has any include/template system. If not, repeat carefully.

## Quality Bar

The final component should feel like a modern AI-native site assistant.

A user should immediately understand:
- “I can ask something here.”
- “This is part of CRBOX.”
- “This will help me navigate faster.”
- “This is smarter and more useful than a normal chat bubble.”

It should not feel like:
- A WhatsApp bubble
- A generic Intercom clone
- A support bot from an old ecommerce site
- A decorative search bar that does nothing

## Done Looks Like

- The old circular orange chat bubble no longer appears on public pages.
- A floating horizontal AI prompt bar appears consistently across public pages.
- Clicking/focusing it expands a polished assistant panel with useful quick prompts.
- Quick prompts guide users to relevant CRBOX pages without fake AI responses.
- The component is responsive, accessible, and visually integrated.
- It does not overlap badly with the back-to-top button.
- It does not break existing forms, calculators, menus, or page layouts.
- No dashboard/private portal pages are affected.
