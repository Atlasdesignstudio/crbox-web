# Fix Miami ZIP code + keep portal secondary navbar fully visible on scroll

## What & Why
There are two targeted fixes needed across the client portal.

### 1. Miami shipping address ZIP code is outdated
A user reported that the Miami shipping address shown in the portal still displays the old ZIP code:

- current incorrect ZIP: 33166
- correct ZIP that should now be shown: 33195

This Miami address is the shared CRBOX shipping address used across client accounts, so this is not a one-off per-user correction. It should be updated consistently wherever that shared Miami address is rendered in the client portal.

### 2. Secondary portal navbar partially hides on scroll
The secondary navbar inside the client portal — the one with:
- Dashboard
- Mis Paquetes
- Mis Facturas
- Cotizaciones
- Mi Cuenta

is currently not behaving correctly while scrolling.

When the user scrolls down, it becomes partially hidden / clipped instead of remaining fully visible.

This should be fixed across all client portal pages.

The requirement is simple:
- keep it fully visible
- keep it fixed/sticky as intended
- do not let it get partially covered
- do not redesign it
- do not change its visual style
- do not alter spacing, hierarchy, or active-tab styling beyond what is strictly necessary to fix the bug

This is a stability / layout fix, not a redesign.

Likely causes may include:
- wrong top offset
- header height mismatch
- stacking context / z-index issue
- sticky container inside an overflow-clipped parent
- page-specific wrapper differences
- incorrect margin/padding compensation below the sticky element

Keep the secondary navbar fully visible during scroll without changing its design.

### Requirements
- maintain the current visual appearance
- maintain current width/alignment
- maintain current tab structure

- ensure it stays stable across all portal pages, not just one

### Important
