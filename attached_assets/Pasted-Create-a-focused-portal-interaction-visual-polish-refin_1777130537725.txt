- it should feel intentional, premium, and readable

- when the user taps Copiar, the button sometimes shows an awkward framed/highlighted state with a different-looking block/background
- this feels broken or dev-like
- the copy interaction should feel clean and polished
- no ugly persistent focus/active artifact should remain after tapping

Please refine that card so:
- the icon block has proper contrast
- the copy CTA looks and behaves cleanly
- the interaction feedback feels intentional, subtle, and polished

### 2. Remove ugly tap/click/focus artifacts across the portal
Across the portal, especially on mobile, when interacting with:
- dropdowns
- filters
- buttons
- selects
- controls

there are visible strokes/outlines/highlight states that feel like browser/dev defaults rather than product-quality interaction design.

These should be audited and cleaned up.

Important:

- instead, replace awkward default-looking states with refined, intentional focus/active treatments aligned with the CRBOX UI

Goal:

This applies especially to:
- filter dropdowns in Mis Paquetes

### 3. Modernize the portal tab navigation aesthetically
The portal tab bar currently works and proportions are mostly correct, but visually it can feel more modern and intentional.

Do not break its structure or usability.
Do not reduce clarity.
But refine its visual treatment so it feels more premium and differentiated from the main navbar above.

Direction:
- keep icon sizes and label proportions sensible
- keep the tab system clear and highly usable
- make the tab navigation feel more like a deliberate secondary navigation component, not just another plain row under the main header

Possible direction:
- a more refined enclosed treatment
- more modern tab containers
- better active-state treatment
- clearer distinction from the main top navbar
- on desktop especially, consider a more centered and composed tab-bar treatment
- a subtle pill / contained / softly elevated treatment may work if done tastefully and within the CRBOX design language

The user specifically likes the idea that the portal tabs could feel:
- more premium
- more modern
- more clearly separated from the top navbar
- more intentionally framed

But this must remain tasteful and consistent with the rest of the portal.

### 4. Mis Facturas expansion is not mobile-native and must be fixed
In Mis Facturas, the expanded single-factura view is currently not designed for mobile at all.

Current problem:
- when expanding a factura, the content opens horizontally
- it is not properly readable on mobile
- it is not naturally scrollable/accessible
- the information is not digestible in that format
- it does not feel mobile-native

This needs to be redesigned within the existing design system into a mobile-native pattern.

Requirements:
- on mobile, expanded factura content must be readable without awkward horizontal overflow
- the information hierarchy must be easy to digest
- if the current expanded table-like structure is too dense for mobile, adapt it into a better mobile pattern such as:
  - stacked sections
  - key-value grouped rows
  - collapsible blocks
  - cardified receipt groupings
  - progressive disclosure

The goal is:
- the factura detail must feel intentionally designed for mobile
- the user must be able to read and understand the included data easily
- no broken sideways layout
- no inaccessible hidden content

Desktop can keep a denser structure if appropriate, but mobile must be truly adapted.

### 5. Remove the edit button from Mis Paquetes
In Mis Paquetes, the edit button should not be there.

Reason:
- this is a visualization / tracking dashboard for the user’s packages

- package data is system-driven / operational, not user-editable here
- leaving the edit action visible is misleading

Please:
- remove the edit button/action from Mis Paquetes on both desktop and mobile
- keep the remaining relevant actions only
- make sure spacing/alignment of the action area still looks intentional after removing it
- do not leave broken gaps or awkward icon balance
