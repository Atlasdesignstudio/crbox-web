Please create a complete explanatory report of how the current CRBOX quote flow works end to end.

This is not a bug audit, debugging task, QA task, or implementation task.

The goal is to understand and document the current quote journey clearly, as a business/product flow, so we can later design a better AI-first quoting experience.

Do not change code.
Do not fix anything.
Do not propose implementation tasks yet.

## Main objective

Explain how the current CRBOX quote system works across all touchpoints, from the moment a customer starts a quote until the quote is reviewed by sales, calculated, sent back to the customer, viewed in the portal, followed up, and reminded if the customer does not respond.

## Touchpoints to include

Map the full journey across:

1. Public Cotizar page
2. Public Calculadora page
3. Product URL “Analizar” flow
4. Client portal Cotizaciones / Solicitudes
5. Internal CRBOX sales/admin quote panel
6. Admin per-product calculator
7. Quote response generation
8. Customer quote response email
9. Client portal quote response view
10. Customer decision / reply / next-step flow, if it exists
11. Quote reminder email
12. Expiration / deadline logic
13. Any quote-related chat or AI assistant behavior

## What the report should explain

For each touchpoint, explain:

- What the user or sales/admin sees
- What action they can take
- What data is collected
- What happens after the action
- What system, endpoint, function, database field, or email template is involved
- What the next step in the journey is
