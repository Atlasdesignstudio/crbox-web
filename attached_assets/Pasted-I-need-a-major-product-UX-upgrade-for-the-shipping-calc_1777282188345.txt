I need a major product/UX upgrade for the shipping calculator, without breaking the current calculation logic that already works well.

## Context
The current calculator is good in two ways:
1. the calculation logic itself is working well
2. the core UI structure is already functional

But the experience is still too manual and not user-friendly enough.

Right now the user has to enter too much information by hand:
- product/content
- weight
- purchase value
- dimensions
- destination

That makes the tool useful, but still harder than it should be.

I want to evolve this into a much more helpful and persuasive calculator experience.

---

# Main product goal
The calculator should not only estimate the cost of one shipment.

It should also help the user understand:
- how consolidated shipping works
- why consolidating multiple packages is usually cheaper
- how much they save by consolidating vs sending separately

That value proposition needs to become obvious inside the calculator.

---

# Core logic that must be reflected
For consolidated packages, the business rule is:

- sum the real weights
- sum the volumetric weights
- charge based on the greater of the two totals
- and make clear that sending consolidated is cheaper because otherwise the user pays freight + handling for each package separately

I need the calculator UX and logic to reflect this clearly.
